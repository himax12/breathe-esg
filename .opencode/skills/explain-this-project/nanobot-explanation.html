Concept 1

Why Does This Exist?

The problem: Building AI agents that can actually do work is incredibly complex. Most frameworks are massive, require extensive setup, and become black boxes.

Think about what an AI agent needs to do:

  • Understand what you want (interpret messages)
  • Use tools (read files, run code, search the web)
  • Remember context (keep track of conversation history)
  • Handle errors gracefully (retry when things fail)
  • Connect to different platforms (Slack, Discord, Telegram...)

Most frameworks bundle all of this together in ways that are hard to understand or customize.

The solution: Nanobot

Nanobot is designed to be ultra-lightweight and readable. The core agent loop is small enough to fit in your head. Yet it supports everything you need for a production AI assistant.

Think of it like a restaurant kitchen:

A big restaurant might have dozens of chefs, each doing tiny specialized tasks. But a small, efficient kitchen can produce amazing food with just a few skilled people.

Nanobot is that small, skilled kitchen — it keeps the core simple while still doing everything complex frameworks do.

The Runner (Tool Execution)

The runner handles the actual tool-calling loop. Let's trace through it:

# Pseudocode for the agent runner
async def run():
    messages = build_prompt()  # System + history + current message
    
    for iteration in range(max_iterations):
        # 1. Call the LLM with current messages
        response = await llm.chat(messages)
        
        # 2. If LLM wants to use a tool...
        if response.tool_calls:
            for tool_call in response.tool_calls:
                result = await execute_tool(tool_call)
                messages.append(result)  # Add tool result to history
            continue  # Keep looping with new info
        
        # 3. If LLM gives a text response, we're done!
        return response.content

Think of it like debugging with a partner:

You tell your partner what you want. They say "I'll need to check a few files." They read the files, come back with questions, read more files, and eventually give you an answer.

Each file read is a "tool call" — the agent gathering information before responding.

Provider Interface

All providers implement the same interface, so you can swap them:

LLMProvider:
    async def chat(messages, tools, model) -> LLMResponse
    async def complete(prompt, ...) -> str

Concept 10

Sessions: Keeping Track of Conversations

Each conversation lives in a Session. Sessions are persisted so the agent can resume after restart:

Session:
    key: str              # "telegram:chat123"
    messages: list        # Full conversation history
    metadata: dict        # Checkpoint info, etc.
    
    # Methods:
    get_history()         # Get all messages
    add_message()         # Add to conversation
    save()                # Persist to disk
    restore()             # Load from disk

Checkpoint System

If the agent crashes mid-turn, it can recover:

CRASH RECOVERY FLOW:

1. Agent starts processing message
       │
2. Save checkpoint (assistant message + tool state)
       │
3. Agent crashes (OOM, SIGKILL, etc.)
       │
4. Agent restarts, loads session
       │
5. Restore checkpoint → replay into messages
       │
6. Continue from where we left off ✓

Think of it like auto-save in video games:

You don't lose progress when the game crashes because it saved automatically. The checkpoint system does the same for the agent.

Concept 11

MCP: Model Context Protocol

Nanobot supports MCP servers, which is a standard way to give AI models additional capabilities:

# In config:
mcp_servers:
  - name: "filesystem"
    command: "npx @modelcontextprotocol/server-filesystem"
  - name: "github"  
    command: "npx @modelcontextprotocol/server-github"

MCP tools appear alongside built-in tools, seamlessly integrated into the agent's toolkit.

Think of it like plugging in a USB device:

You connect a new device (MCP server), and it immediately works with your computer (Nanobot). The agent can use new tools without any extra configuration.

Quick Start

Try It Yourself

Here's how to run nanobot:

# Install
pip install nanobot-ai

# Configure (creates .nanobot.yaml)
nanobot setup

# Run
nanobot

# Or with Docker
docker run -v ~/.nanobot:/app/.nanobot ghiku/nanobot

Configuration (./nanobot.yaml)

provider:
  type: anthropic  # or openai, deepseek, etc.
  model: claude-sonnet-4-20250514
  
channels:
  telegram:
    bot_token: "your-token"
  
tools:
  exec:
